About One-Page Apps

onePageApps.gic.mx is a curated library of Cloudflare Worker templates you can browse, preview, remix, generate with AI, and publish without standing up a traditional backend.

visibility

Preview-first workflow

Every app is discoverable through browse and preview flows so you can inspect the experience before copying or deploying anything.

auto_awesome

AI-assisted generation

The `generate.html` workflow sends prompts to a private Cloudflare bridge worker that returns deployable Worker JavaScript and validation feedback.

rocket_launch

Deploy your way

Use your own Cloudflare account for free, or publish to a hosted `*.onepageapps.gic.mx` subdomain once your plan allows it.

How the platform is structured

Public repo: this static site hosts the catalog, previews, category pages, and the client-side generator/deploy UI.

Private repo: `onePageAppsPrivate` contains the Cloudflare Worker that handles AI generation, validation, Stripe checkout verification, hosted deploys, and API key management.

Companion app: the iOS/macOS app syncs from the same `docs/apps-index.json` manifest, so web and native discovery stay aligned.

Pricing and access

Same structure used across the PRD: free browsing and BYO deploys, then paid hosted features.

API and bridge routes

The public site talks to the private bridge Worker. These routes are the core ones exposed in the PRD and implemented in the companion repo:

Generation

  • `POST /ai/create`
  • `POST /ai/create/stream`
  • `POST /ai/edit`
  • `POST /ai/edit/stream`
  • `POST /validate`

Deploy + billing

  • `POST /deploy/byo`
  • `POST /deploy/gic`
  • `POST /apps/list`
  • `POST /apps/delete`
  • `POST /stripe/*`

API keys

  • `POST /api/keys/create`
  • `POST /api/keys/list`
  • `POST /api/keys/revoke`

Admin gift keys

  • `POST /admin/keys/create`
  • `POST /admin/keys/list`
  • `POST /admin/keys/revoke`
Browser calls are CORS-restricted to `https://onepageapps.gic.mx`, while API-key routes are designed for server-side or automation use. JWT auth unlocks paid features, and BYO credentials stay on the client when possible.

Built by GIC

This site is a companion to the OneTimeUseWebApp ecosystem and follows the same public/private split used by `forms.gic.mx`: public browsing and UI in one repo, protected worker logic in a separate private repo.
